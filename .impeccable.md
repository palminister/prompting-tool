## Design Context

### Users
Prompt Tool is for prompt engineers, product engineers building agent workflows, AI prototypers, and adjacent technical collaborators such as designers, PMs, and technical writers. They use it during focused desktop authoring sessions to turn raw JSON payloads into reusable prompt variables, then assemble clean markdown deliverables without copy-paste errors.

### Brand Personality
Dependable, lucid, exacting.

The interface should feel calm, capable, and sharply organized. It should support deep work rather than perform for attention. Interactions should feel lightweight and immediate, with high legibility, disciplined spacing, and a strong sense of order.

### Aesthetic Direction
Light mode only.

Reference direction:
- Notion: crisp hierarchy, disciplined neutral palette, low-friction productivity patterns
- Linear docs/editorial tooling: modern typography, compact precision, quiet confidence

The product should feel like a serious writing workspace with a predominantly black, white, and gray palette. Accent color should be used sparingly and functionally, not as atmosphere. Typography should feel modern, clean, and highly readable. The product should avoid cream tones, soft warmth, decorative gradients, novelty-heavy AI styling, and dashboard-like visual noise.

### Design Principles
1. Clarity over decoration: hierarchy, labeling, and spacing should make the workflow self-evident.
2. Neutral by default: the interface should lean black, white, and gray, with accent reserved for meaningful highlights.
3. Writing flow first: controls should stay out of the way until needed, and authoring should remain the center of gravity.
4. System over one-offs: use design tokens, shared components, and repeatable interaction patterns instead of bespoke styling.
5. Tidy precision: spacing, borders, and typography should feel neat, compact, and intentional.
